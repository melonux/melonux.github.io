摘要

我将保罗·哈尔莫斯 (Paul R. Halmos) 文中提到的写作注意事项提炼为一系列明确的条款，并补充具体的正反面示例，以便快速掌握和应用。

引言

著名数学家 Paul R. Halmos 撰写的《How to write mathematics》是一篇极具洞察力和实践指导意义的文章。它不仅为数学领域的写作者提供了宝贵的建议，其核心思想也适用于任何需要清晰、严谨和有效沟通的写作领域。

一、 战略与规划 (Big Picture Strategy)

在动笔之前，必须明确写作的宏观方向。

  • 条款1：确保主题单一且明确。

    • 说明： 一篇文章只能有一个核心思想。 避免内容空洞或试图在一篇文章里塞进太多不相干的主题。

    • 错误示例 ❌： 写一本叫《微积分》的书，因为它实际上是逻辑、几何、分析、代数等多个主题的混合体，难以写好。

    • 正确示例 ✅： 写一篇论文，证明“有界调和函数必为常数”，主题清晰、聚焦。 或者写一本书，专门探讨“度量空间中的紧集理论”。

  • 条款2：为具体的读者而写。

    • 说明： 在脑海中锁定一个清晰的读者画像（例如，一位刚学完某门课程的学生），并时刻站在他的角度思考，预判他可能遇到的困难并加以解决。

    • 错误示例 ❌： “本文适合所有对拓扑学感兴趣的读者。”（过于宽泛，无法提供有效指导）

    • 正确示例 ✅： “我的目标读者是一位刚学完点集拓扑，正要开始学习代数拓扑的研究生。他可能对‘同伦’的直观概念很模糊，所以我会用大量图形和形变的例子来引入，然后再给出严格定义。”

  • 条款3：先组织，再动笔。

    • 说明： 写作的核心贡献在于组织材料。 围绕核心的、关键的示例和反例来构建你的文章结构，而不是简单地罗列定义和定理。 逻辑结构比线性顺序更重要。

    • 错误示例 ❌： 按照“定义-定理-证明”的枯燥顺序线性罗列所有知识点。

    • 正确示例 ✅： 在介绍“群”的概念时，不直接下定义，而是先从“正方形的对称性”这个核心例子出发，引出变换、封闭性、单位元、逆等概念，最后再提炼出抽象的群公理。

  • 条款4：提前规划符号系统。

    • 说明： 在写作初期就设计好全文的符号，保持一致性与和谐感，避免临时起意导致混乱。

    • 错误示例 ❌： 在一句话中，用 \epsilon 表示一个小数，在下一段又用它表示集合的“属于”关系（\in）。 或者，写出 a x_1 + b x_2 这种索引和字母不协调的表达式。

    • 正确示例 ✅： 事先规定：小写希腊字母 α,β,γ 代表标量；大写字母 A,B,C 代表矩阵；草书体 \mathcal{F},\mathcal{G} 代表函数集合。并在全文中严格遵守。

二、 写作与修改流程 (Writing & Revision Process)

写作是一个动态的、持续改进的过程。

  • 条款5：采用螺旋式写作。

    • 说明： 写作并非线性过程。写完后续章节后，要根据新的理解和需要，回头重写前面的章节。 这是一个不断迭代、完善的过程。

    • 示例 ✅： 写完第三章“线性变换”后，发现第一章“向量空间”的例子（如多项式空间）不够充分，无法很好地展示“核”与“像”的概念。因此，回到第一章，增加或修改例子，使其能更好地服务于第三章的内容。

  • 条款6：重写，而非编辑。

    • 说明： 当草稿需要大改时，不要在原文上用红笔涂涂改改。 最有效的方法是把每个字都重新写一遍，这能迫使你摆脱对原有文字的路径依赖，做出更彻底的改进。

    • 错误示例 ❌： 在打印出的草稿上画箭头、删改词语、在页边空白处添加大段文字。

    • 正确示例 ✅： 打开一个空白文档，参照草稿的思路和内容，用全新的句子和结构重新将思想表达一遍。

  • 条款7：坚持每日写作并“启动水泵”。

    • 说明： 成功的关键在于每天持续不断地写一点。 为了克服第二天的写作惰性，在每天收工前，为第二天的工作做好准备，比如写下第二天的第一个句子或定好下一节的标题。

    • 示例 ✅： 今天写完了“紧集”的定义和性质。在停止工作前，我写下“下一节：有限覆盖引理”这个标题，并写下第一句话：“为了证明紧集上的连续函数必有界，我们首先需要一个强大的工具，即有限覆盖引理。”

三、 数学内容的呈现 (Presenting Mathematical Content)

如何清晰、诚实地表达数学思想。

  • 条款8：先陈述定理，再进行证明。

    • 说明： 不要长篇大论之后，突然说“因此我们证明了……”。 应该先把定理的陈述（包括所有假设和结论）清晰地展示出来，让读者知道目标是什么，然后再开始证明。

    • 错误示例 ❌： “考虑一个函数 f…… 它在闭区间 [a,b] 上连续……取任意 \epsilon>0 ……经过一系列推导……我们发现 f 在该区间上一致连续。因此我们证明了 定理3.1……”

    • 正确示例 ✅： “定理3.1 (Heine-Cantor 定理): 若函数 f 在闭区间 [a,b] 上连续，则 f 在该区间上一致连续。 证明：……”

  • 条款9：保持陈述的纯粹性，剔除无关信息。

    • 说明： 在定理或定义的陈述中，不要加入多余的、不必要的假设或解释性语句。

    • 错误示例 ❌： “定理： 对于一个交换半单环 R（不失一般性，我们可以假设它有单位元），对任意 x,y \in R，我们有 x^2-y^2=(x−y)(x+y)。”（交换性是必要的，但半单和单位元是不必要的假设，会误导读者 ）。

    • 正确示例 ✅： “命题： 在任意交换环中，对任意元素 x,y ，都有 x^2-y^2=(x-y)(x+y) 。”

  • 条款10：诚实地对待“显而易见”。

    • 说明： 使用“显然”、“易证”等词语时，必须确保它对目标读者来说确实如此，并且你自己能在几个月后依然觉得它显然。 更重要的是，确保它是对的。 如果一个结论没有给出证明，要向读者坦白它的状态（例如，“其证明超出了本书范围，读者可参考文献[X]”）。

    • 错误示例 ❌： “由引理1，显然可得 p 蕴含 q。”（实际上中间需要三步不那么明显的推导，作者因为自己熟悉就跳过了）。

    • 正确示例 ✅： “结合引理1和简单的代数运算，我们可以直接验证 p 蕴含 q。我们将这个验证过程留给感兴趣的读者作为练习。”

  • 条款11：不要重复证明，应提炼引理。

    • 说明： 如果在证明定理2时，发现很多步骤和定理1的证明“几乎一样”，这强烈暗示着背后有一个更根本的、共同的引理。 你应该花时间找到这个引理，单独证明它，然后用它来简化两个定理的证明。

    • 错误示例 ❌： 证明定理2时写道：“这里的步骤与定理1的证明完全类似，故略。”

    • 正确示例 ✅： “引理A： [陈述引理]。证明： [证明引理]。 定理1的证明： 由引理A，可得…… 定理2的证明： 由引理A，亦可得……”

四、 语言与风格 (Language and Style)

文字本身是传达思想的工具，必须打磨。

  • 条款12：永远不要使用“any”。

    • 说明： 在数学语境中，“any”一词极其模糊，可以表示“任意一个”（全称量词 \forall）或“存在一个”（存在量词 \exist）。 为避免歧义，请用“each”、“every”或“all”代替全称量词，或用“some”或“there exists”表达存在。

    • 错误示例 ❌： “Prove that any complex number is a product of…”

    • 正确示例 ✅： “Prove that every complex number is a product of…” 或 “Show that for each complex number z, …”

  • 条款13：精确使用技术术语。

    • 说明： 尊重数学术语的严格定义。 例如，区分“函数”和“函数值”，“序列”和“可数集”。

    • 错误示例 ❌： “函数 x^2+1 是一个偶函数。” ，“可测集序列的并集是可测的。”

    • 正确示例 ✅： “由 f(x)=x^2 +1 定义的函数 f 是一个偶函数。” “一个可数族可测集的并集是可测的。”

  • 条款14：克制地、有意义地使用“我们(we)”。

    • 说明： “我们”应该指“作者和读者”这个共同体。 用于引导读者进行共同的逻辑推理。不要用它来指代作者自己（单数）或表达与读者无关的个人事务。

    • 错误示例 ❌： “我们 (其实是我) 在1998年发现了这个定理。” “我们感谢我们的妻子……”

    • 正确示例 ✅： “利用引理2，我们现在可以证明定理3。”

五、 符号与版面 (Symbols and Layout)

符号和页面外观是无声的语言。

  • 条款15：抵制不必要的符号和字母。

    • 说明： 能用文字说清楚的，就不要引入符号。 不要使用只出现一次，没有实际贡献的“一次性”字母。

    • 错误示例 ❌： “在紧空间上，每个实值连续函数 f 都是有界的。” (这里的 f 是不必要的，它没有在别处被引用)。

    • 正确示例 ✅： “在紧空间上，所有实值连续函数都是有界的。”

  • 条款16：永远不要用符号开始一个句子。

    • 说明： 这会使标点符号（尤其是句号）被过度使用，容易让读者忽略句子的分隔，造成阅读困难。

    • 错误示例 ❌： “X is a compact set. X has the Heine-Borel property.”

    • 正确示例 ✅： “The set X is compact. It follows that X has the Heine-Borel property.”

  • 条款17：像标点普通句子一样标点数学表达式。

    • 说明： 一个数学公式或不等式在句子中扮演着一个子句的角色，因此它的前后需要正确的标点符号。

    • 错误示例 ❌： “We see that x>0 therefore the function is positive.”

    • 正确示例 ✅： “We see that x>0, and therefore the function is positive.”

  • 条款18：明智地使用布局和展示。

    • 说明： 避免整页都是密集的文字（令人生畏）或满篇都是符号（让人害怕）。 适当地将关键公式、定义或定理独立成行并编号，以打散文本，帮助读者抓住重点。 但不要给无关紧要的公式编号。

六、 最终阶段 (Finishing Touches)

写作的最后环节同样重要。

  • 条款19：坚决捍卫你的数学表达。

    • 说明： 你必须准备好与编辑或校对员沟通。 如果他们为了遵循通用的、非数学的风格规则而要修改你的精确数学表达，导致含义错误或模糊，你必须坚持己见。

    • 示例 ✅： 一位编辑将你的“measure-preserving transformation”（保测变换）改成了“nonmeasure preserving transformation”，这在数学上是无稽之谈。你必须坚决将其改回。

  • 条款20：学会停止。

    • 说明： 写作永远没有完美的终点。 总会有可以改进的地方。 在完成所有你能想到的内容、检查和校对、并让文稿“沉淀”一段时间后，就必须果断地停下来。

    • 行动指南 ✅： 完成草稿 \rightarrow 校对计算 \rightarrow 放置数月使其“成熟” \rightarrow 最后通读一遍 \rightarrow 停止并提交。
